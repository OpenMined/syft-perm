Quick Start Guide

Learn SyftPerm essentials in 5 minutes. Master file permission management for collaborative data science.

üìã Prerequisites: Make sure you have installed SyftPerm before starting this guide.

Step 1: Open Your First File

SyftPerm works with any file or folder on your system. Let's start with the basics:

import syft_perm # Open a file with SyftPerm (creates if it doesn't exist) file = syft_perm.open("my_data.txt") # Check current permissions print(file) # Shows permission table
‚ú® What happened? SyftPerm opened your file and showed you that it currently has no permissions set. The file is essentially private by default.

Step 2: Grant Your First Permissions

Let's give some users access to your file:

# Grant different permission levels file.grant_read_access("reviewer@university.edu") file.grant_write_access("colleague@company.com") file.grant_admin_access("boss@company.com") # See the results print(file)

You'll see a beautiful table showing:

  • reviewer@university.edu - Can read the file
  • colleague@company.com - Can read + write (write includes read)
  • boss@company.com - Can do everything (admin includes all permissions)
üîë Key Insight: Permissions are hierarchical. Write permission automatically includes read permission. Admin includes everything.

Step 3: Make Data Public

Want to share data with everyone? Use the special * user:

# Make the file publicly readable file.grant_read_access("*") # Now anyone can read it print(f"Random user can read: {file.has_read_access('anyone@anywhere.com')}") # Output: True
‚ö†Ô∏è Security Note: Public access means anyone with the file path can read it. Use this for published datasets, documentation, or truly public information only.

Step 4: Manage Entire Folders

The real power comes from setting permissions on folders. All files inside automatically inherit:

# Create a project folder and grant permissions project = syft_perm.open("my_project/") project.grant_read_access("team@company.com") project.grant_write_access("leads@company.com") # ALL files in the folder now have these permissions! # Even files that don't exist yet will inherit them when created.

This is incredibly powerful for large projects:

  • Set permissions once on the project folder
  • Hundreds of files automatically get the right access
  • New files inherit permissions automatically
  • No need to manage individual file permissions

Step 5: Use Powerful Patterns

Control specific file types with glob patterns:

# Give developers access to all Python files project.grant_write_access("*.py", "dev@company.com") # Give writers access to all documentation project.grant_write_access("docs/**/*.md", "writers@company.com") # Give data scientists access to datasets project.grant_read_access("data/**/*.csv", "datascience@company.com")

Pattern examples:

Common Patterns

  • *.py - All Python files
  • *.md - All Markdown files
  • test_*.py - All test files
  • config.* - Config files

Advanced Patterns

  • docs/** - Everything under docs/
  • **/*.json - JSON files anywhere
  • src/**/test*.py - Tests in src/ tree
  • data/*/processed/*.csv - Specific structure

Step 6: Debug and Verify Access

Never guess why permissions work or don't work:

# Check if a user has specific access can_read = file.has_read_access("user@example.com") can_write = file.has_write_access("user@example.com") # Get detailed explanation of WHY permissions were granted/denied explanation = file.explain_permissions("user@example.com") print(explanation)

The explanation will show you exactly:

  • Which pattern matched (if any)
  • Which directory provided the permissions
  • Why access was granted or denied
  • The inheritance chain that was followed
üîç Pro Tip: Use explain_permissions() whenever you're confused about why someone does or doesn't have access. It's your debugging superpower!

Step 7: Use the Visual Editor (Optional)

For team members who prefer GUIs, SyftPerm includes a Google Drive-style web editor:

# Get the web editor URL for any file or folder editor_url = syft_perm.get_editor_url("my_project/") print(f"Edit permissions at: {editor_url}") # Open in browser - shows a beautiful permission management interface

The web editor provides:

  • Point-and-click permission management
  • User autocomplete from your SyftBox datasites
  • Visual file compliance checking
  • Real-time permission preview
