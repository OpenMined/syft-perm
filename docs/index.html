Installation

Install SyftPerm via pip in just one command:

pip install syft-perm
💡 Pro Tip: Install with optional dependencies for the full experience:
pip install "syft-perm[server,display]"

Key Features

🎯 Intuitive Permission Hierarchy

Four permission levels (Read → Create → Write → Admin) that build naturally on each other. Higher permissions automatically include lower ones.

file.grant_write_access("alice@example.com") # Alice automatically gets read + create + write

🌟 Powerful Pattern Matching

Use glob patterns to control hundreds of files with single rules. Supports wildcards, recursive matching, and intelligent specificity resolution.

# All Python files anywhere in the project folder.grant_access("*.py", "dev@team.com") # All docs, nested arbitrarily deep folder.grant_access("docs/**/*.md", "writers@team.com")

🔍 Nearest-Node Algorithm

Predictable inheritance that finds the "closest" permission rules. No confusing accumulation - the nearest directory with matching rules wins.

# Files inherit from nearest parent with rules project/ ├── data/ ← permissions set here │ └── analysis/ │ └── results.csv ← inherits from data/

🐛 Built-in Debugging

Never guess why permissions work or don't work. Trace the exact reasoning behind every access decision.

file.explain_permissions("alice@example.com") # Shows: "Read access granted by pattern '*.py' # in directory /project/src"

📁 Folder-Level Efficiency

Set permissions once on directories, and all files inside automatically inherit them. Massive time savings for large projects.

folder = syft_perm.open("my_project/") folder.grant_read_access("team@company.com") # All 500+ files in project get team access!

🎮 Interactive Permission Editor

Google Drive-style web interface for managing permissions. Perfect for non-technical team members.

import syft_perm url = syft_perm.get_editor_url("my_file.txt") # Opens web interface for permission management
